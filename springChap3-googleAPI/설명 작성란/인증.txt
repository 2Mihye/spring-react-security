
OAuth (Open Authorization)
	사용자가 인증을 할 수 있도록 제 3자에게 허용해준다는 프로토콜
	웹이나 모바일에서 사용자의 데이터에 대한 인증이나 권한을 부여할 수 있도록 해주는 역할
	
	1. 인증(Authentication) : 사용자를 인증하고, 클라이언트가 프로그램에서 프로그램을 사용할 수 있도록 허용 또는 거부하는 과정을 나타냄.
	
	2. 권한부여(Authorization) : 사용자가 프로그램에 대한 특정 작업을 할 수 있도록 권한 부여 (특정 작업이란 프로그램을 만든 사람이 만든 역할에 대한 작업을 말함)
	
	3. 토큰(Token) : 권한이 부여된 사용자에게는 특정 권한을 나타낼 수 있는 토큰을 생성해서 제공하고 관리	
			* Access Token
				사용자 데이터에 허용하는 데 사용하는 토큰으로 주로 클라이언트와 서버 간에 통신에서 주로 사용하고 클라이언트의 요청에 대한 권한을 나타내며, 유효기간이 존재함.
			* Refresh Token
				Access Token이 만료된 경우, 리프레시 토큰을 사용하여 새로운 토큰을 얻음.

OAuth : 첫 번째 버전			
		서명으로 진행하다보니 좀 더 복잡한 프로세스를 가지고있음.
			
OAuth 2.0 : 두 번째 버전(더 간결함)
			http 쪽에서 흐를 수 있도록 되어있기 때문에 구현이 쉽고 간단
			
			1. Client(클라이언트) : 프로그램에 접근할 수 있도록 허용하게 되면 클라이언트는 접근이 가능
			2. Owner(오너) : 데이터에 대한 권한을 부여하거나 거부하는 역할
			3. Authorization(권한) : 권한을 부여하고 토큰을 발급하는 역할
			4. Server(서버) : 보호되는 데이터를 가지고있고, 토큰이 유효한지 확인 후 클라이언트에게 정보를 제공
