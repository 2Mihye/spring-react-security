
* HTTP (Hyper Text Transfer Protocol)
	HTTP는 클라이언트와 서버 간의 통신을 위한 프로토콜로 주로 웹에서 데이터를 주고 받을 때 사용한다.
	HTTP는 메서드, 헤더, 바디로 이루어져있다.
		메서드 : GET, POST
				GET : 주로 API나 DB에서 데이터를 가지고 오는 데 사용한다.
				POST: DB나 서버에 데이터를 제출하고 싶을 때 사용한다.
		헤더(Header) : 인증 토큰이나 캐시, 사용자 정보와 같은 내용은 meta나 데이터 헤더 부분을 이용하여 전달한다.
					  눈 앞에서 직접적으로 보이지는 않지만 중요한 정보를 담을 때 많이 사용한다.
		바디(Body) : 주로 <input/>이나 <text/>, <textarea/>로 바디에 작성하는 태그 내용을 DB에 실제로 전송되는 데이터를 포함한다.
		
		
* RESTful
	REST 아키텍처 스타일을 따르는 웹 서비스를 말한다.
	자원(Resource) : URL로 나타내는 주소에 담겨있는 모든 정보들.
				    예를들어 /member는 회원 정보가 들어있을 수 있고, 회원의 정보 외에 다른 내용이 있음과 상관없이 우리가 URL에 담고자하는 내용이 들어있다.
	표현(Representation) : URL로 표현하는 자원이 다양한 방식으로 표현될 수 있다.
						  JSON, XML 등이 있지만 위 둘이 대표적인 표현법.
	상태전이(Stateless) : 각 요청을 서버는 저장하지 않음. 서버는 서버이고 각각의 요청은 각각의 요청에 담아 정보를 표현해야함. 간단히 말해 /user는 user정보만 /product는 product 정보만
	통합 인터페이스(Uniform Interface) : 인터페이스는 개발자가 지정해준 디자인을 가지고 있으며, 컴퓨터가 임의적으로 데이터를 변경하지 않고 일관성있게 화면 데이터를 제공하는 것.
